# 团队开发规范

## 项目概述

- **项目名称**: 念铭导航
- **技术栈**: Vue 3 + Element Plus + Pinia
- **构建工具**: Vite

## 开发规范

### 代码风格

1. **组件命名**
   - 使用 PascalCase（如 `SiteCard.vue`）
   - 组件文件名与组件名保持一致

2. **变量命名**
   - 变量/函数: camelCase
   - 常量: UPPER_SNAKE_CASE
   - 布尔值: 使用 `is`、`has`、`should` 前缀

3. **CSS 规范**
   - 使用 CSS 变量管理主题色
   - 优先使用 `rem` 单位
   - 过渡动画使用 `cubic-bezier(0.4, 0, 0.2, 1)`

### 提交规范

**提交前必须确认，禁止自行提交代码**

提交信息格式:
```
<type>: <description>

[可选的详细描述]
```

类型说明:
- `feat`: 新功能
- `fix`: 修复 bug
- `style`: 样式调整
- `refactor`: 重构
- `docs`: 文档更新

### 开发流程

1. 修改前确认需求
2. 编写代码并自测
3. 提交前与负责人确认
4. 确认后执行提交

### 文件结构

```
src/
├── components/      # 组件
│   ├── layout/      # 布局组件
│   ├── ui/          # UI 组件
│   └── admin/       # 管理后台组件
├── views/           # 页面视图
├── stores/          # Pinia 状态管理
├── hooks/           # 组合式函数
├── utils/           # 工具函数
└── assets/          # 静态资源
```

### 注意事项

- 不要添加未要求的扩展功能
- 保持代码简洁，避免过度设计
- 优先使用项目已有的工具和库
- 修改后确保功能正常
