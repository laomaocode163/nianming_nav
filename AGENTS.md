# AGENTS.md ‚Äî nianming_nav

Personal navigation/start-page SPA (Vue 3 + Vite). Navigation content is static JSON; there is **no backend** for site data.

## Commands

- `npm run dev` ‚Äî dev server on port 5173, auto-opens browser
- `npm run build` ‚Äî production build ‚Üí `dist/`
- `npm run fetch-favicons` ‚Äî download all site favicons to `public/favicons/` and regenerate `src/config/faviconManifest.json`. **Run this after adding/editing links** (or pulling new ones) so icons are served same-origin/offline. Use `--force` to re-download all.
- `npm run lint` ‚Äî ESLint + Prettier with `--fix`; `npm run lint:check` for check-only
- `npm run typecheck` ‚Äî `vue-tsc --noEmit` strict type check (run in CI and before `build`)
- `npm run test` ‚Äî Vitest in watch mode; `npm run test:run` for single-run (CI/commits); `npm run test:coverage` for coverage
- `npx vitest run tests/<path>` ‚Äî run a single test file
- `npm run commit` ‚Äî Commitizen conventional-commit prompt (use this instead of plain `git commit`)

**CI** (`.github/workflows/ci.yml`, Node 22) runs in order: `npm ci` ‚Üí `npm run lint:check` ‚Üí `npm run typecheck` ‚Üí `npx vitest run` ‚Üí `npm run build`. It does **not** auto-deploy; `dist/` is deployed to Cloudflare Pages manually (see Deployment).
The husky pre-commit hook runs `npx lint-staged` + `npx vitest run`, so **every commit triggers lint-staged (ESLint/Prettier auto-fix) and the full Vitest suite** ‚Äî keep tests green before committing.

## Conventions / gotchas

- **Auto-imports**: Vue APIs (`ref`, `computed`, ‚Ä¶) are auto-imported via `unplugin-auto-import` and declarations live in `src/auto-imports.d.ts`. App SFCs under `src/components` are auto-registered as global components via `unplugin-vue-components` (declarations in `src/components.d.ts`) ‚Äî do **not** add explicit component imports for them. **No UI library**: Element Plus has been fully removed; `el-input`/`el-button`/`el-icon` and `ElMessage` are replaced by native elements and a lightweight `useToast` composable (`src/composables/useToast.ts` + `ToastHost.vue`). Do not reintroduce Element Plus.
- **Path alias** `@` ‚Üí `src/` (configured in `vite.config.ts`).
- **Styling**: Tailwind CSS **v4** with **CSS-based config** in `src/assets/main.css`. Theme colors are HSL CSS variables (`--color-primary`, `--color-secondary`). There is **no `tailwind.config.js`** ‚Äî do not create one. Use scoped styles in SFCs.
- **TypeScript is strict** (`noUnusedLocals`, `noUnusedParameters` in `tsconfig.json`): unused vars break `build`/`tsc`. `npm run typecheck` (`vue-tsc --noEmit`) enforces it in CI and before `build`. ESLint errors on `any` and unused vars.
- **Edit site content** by changing JSON in `src/config/data/` (categories, links, search, settings); aggregation + Zod validation happen lazily in `src/config/loadConfig.ts` (`loadSiteConfig()`), so Zod/JSON stay out of the first-load bundle. There is **no database**; data is committed JSON.
- **Dev-only Admin UI**: a visual editor for links/categories lives at `/admin` and is registered **only when `import.meta.env.DEV` is true** (route in `src/router/index.ts`, entry from `MainHeader.vue` via a dev-only button navigating to `/admin`, persistence middleware `plugins/devAdminApi.ts`). It writes directly to the `src/config/data/*.json` files, so edits are real source-of-truth changes ‚Äî but it is unreachable in production builds.
- **localStorage** holds theme preference and a favicon URL cache (used to skip known-broken icon sources) ‚Äî never core navigation data.
- **Favicons**: runtime priority is **local same-origin files** in `public/favicons/` (listed in `src/config/faviconManifest.json`, generated by `npm run fetch-favicons`), falling back to the site's own `/favicon.ico`, then the aggregator chain in `src/services/faviconService.ts` (`getFaviconFallbacks`), then a placeholder. Both `public/favicons/` and `faviconManifest.json` are **committed** so production (Cloudflare) serves icons offline. After changing links, run `npm run fetch-favicons` to refresh.
- Component style: Composition API `<script setup>` + TypeScript throughout.

## Architecture

- Single route `/` ‚Üí `HomeView`. Layout = Sidebar (category nav) + MainHeader (search, time, theme toggle) + site-card grid.
- Data flow: JSON ‚Üí lazy loader `src/config/loadConfig.ts` (`loadSiteConfig()`, Zod-validated incl. categoryId/subCategoryId referential integrity) ‚Üí Pinia `data` store (`src/stores/data.ts`, async `init()` populates static data + getters) and `ui` store (`src/stores/ui.ts`, interaction state: selected category, search query/mode, sidebar, pagination) ‚Üí components. Theme is managed by `src/stores/theme.ts`. `App.vue` gates render on `dataStore.init()` + `settingsStore.init()` behind a splash skeleton.
- Favicons are resolved by `src/services/faviconService.ts` with an in-memory + localStorage cache (versioned key, 30-day TTL, 200-entry eviction) and a fallback chain (`favicon.im` ‚Üí `icon.horse` ‚Üí `faviconextractor` ‚Üí DuckDuckGo ‚Üí Google S2). The primary source is the site's own `/favicon.ico` (after the local same-origin files above).
- Runtime settings are applied by `src/stores/settings.ts`: `settings.json` `accentColor` ‚Üí primary HSL CSS variables, and `background` (type `default`/`solid`/`gradient`/`image` + `showBlobs`) ‚Üí `--app-bg-color` / `--app-bg-image` / `--app-bg-fit` CSS variables consumed by `App.vue`. This closes the config ‚Üí UI loop. Background is edited in the admin **ËÉåÊôØ** tab (`AdminSettings.vue`). (Card-style settings are **not** implemented.)

## Deployment

Cloudflare Pages: build command `npm run build`, output directory `dist/`, deployed manually from `main` (CI only builds/validates; it does not hold deploy credentials). (`functions/api/` exists for Cloudflare Functions but is currently empty.)
