## 第 1 章 产品简介与适用对象

### 1.1 产品简介

**念铭导航** 是一个纯前端的个人导航 / 起始页单页应用（SPA）。它将常用网站按分类组织，提供站内搜索、网络搜索、收藏、最近访问、主题切换等能力，适合作为浏览器主页或团队内部导航站使用。

核心特征：

- **无后端**：导航内容全部来自静态 JSON 文件（`src/config/data/`），无需数据库，部署简单。
- **数据驱动**：分类、链接、搜索引擎、站点主题色均由 JSON 配置，改文件即改站点。
- **本地个性化**：收藏、最近访问、主题偏好、密度设置仅保存在用户本机浏览器（localStorage），不影响其他用户。
- **离线图标**：站点图标优先使用仓库内同域文件（`public/favicons/`），可离线 / 同源访问。

### 1.2 适用对象

| 对象 | 关注章节 | 典型诉求 |
| --- | --- | --- |
| **终端使用者**（访问导航站的人） | 第 3 章 | 快速找到网站、搜索、收藏常用站点、切换明暗主题 |
| **维护者 / 部署者**（编辑内容、部署站点的人） | 第 4 章 | 增删网站与分类、配置搜索引擎、获取图标、部署上线 |
| **开发者**（二次开发） | 第 4 章 + 各源码索引 | 理解架构、扩展功能、本地调试 |

## 第 2 章 快速开始

### 2.1 本地运行（开发模式）

```bash
npm install        # 安装依赖
npm run dev        # 启动开发服务器，默认 http://localhost:5173 并自动打开浏览器
```

开发模式下，页面右上角会出现「管理后台」入口（`/admin`），可用于可视化编辑数据。

### 2.2 生产构建与预览

```bash
npm run build      # 生产构建，产物输出到 dist/
npm run preview    # 本地预览构建产物（可选）
```

### 2.3 部署上线

构建产物 `dist/` 部署到任意静态托管（如 Cloudflare Pages）。详见 [4.6 部署到 Cloudflare Pages](#46-部署到-cloudflare-pages)。

> 注意：生产构建中 `/admin` 管理后台会被自动剔除（仅 `import.meta.env.DEV` 时注册），线上无法访问，避免数据被随意修改。

## 第 3 章 终端使用者功能手册

本章面向**访问导航站的最终用户**，描述所有可在界面上直接操作的功 能。

### 3.1 界面布局总览

首页（`/`）采用三区布局：

```
┌───────────────┬───────────────────────────────────────────┐
│               │  顶部头部 MainHeader                        │
│   侧边栏       │   [时间日期] [搜索框 + 模式切换] [偏好][主题] │
│   Sidebar      ├───────────────────────────────────────────┤
│               │  分类标题栏（分类名 + “N 个网站”）           │
│  - 我的常用    │  ┌────┐ ┌────┐ ┌────┐ ┌────┐              │
│  - 最近访问    │  │卡片│ │卡片│ │卡片│ │卡片│  …… 网站卡片网格 │
│  - 人工智能    │  └────┘ └────┘ └────┘ └────┘              │
│  - 开发工具    │                                            │
│  ……            │  底部分页 Pagination                        │
└───────────────┴───────────────────────────────────────────┘
```

- **侧边栏**：分类导航，桌面端可折叠；移动端为抽屉式。
- **顶部头部**：时间日期（仅桌面端）、搜索框、偏好按钮、主题切换按钮；开发模式下还有管理后台按钮；移动端显示汉堡菜单。
- **主区**：分类标题栏 + 网站卡片网格 + 分页。
- **列数自适应**：根据窗口宽度与侧边栏状态估算（移动 / 平板约 2 列，桌面按约 240px 卡片宽度估算，最多 6 列）。
- **每页数量自适应**：根据窗口高度动态推算。

### 3.2 分类导航（侧边栏）

#### 3.2.1 虚拟分类

| 名称 | 标识 | 说明 |
| --- | --- | --- |
| 我的常用 | `__favorites` | 展示用户收藏的网站（见 3.5 收藏） |
| 最近访问 | `__recent` | 展示用户最近点击打开过的网站（按时间倒序） |

> 这两个分类的内容完全来自本机浏览器数据，不同用户看到的内容不同。

#### 3.2.2 一级分类与二级分类

- 点击一级分类（如「人工智能」）进入该分类全部网站。
- 含二级分类的一级分类可**展开 / 收起**（手风琴交互：同一时刻仅展开一个一级分类）。展开后点击二级分类（如「工具教程」）可进一步过滤。
- 点击顶部 Logo（「念铭导航」+ 签名图）返回「全部网站」视图。

#### 3.2.3 侧边栏折叠（桌面端）

- 桌面端可折叠侧边栏，折叠后仅显示图标（宽度约 72px），为卡片区腾出更多横向空间。
- 折叠状态下，二级分类标签会移动到主区顶部以 `SubCategoryMenu` 形式展示（自动换行布局，带计数徽标）。

#### 3.2.4 移动端抽屉

- 移动端侧边栏为抽屉式，点击汉堡菜单打开；点击任一分类项后抽屉自动关闭。

### 3.3 搜索功能

搜索框位于顶部头部中央，支持两种模式。

#### 3.3.1 模式切换

- **站内搜索（默认）**：图标为放大镜（`Search`）。输入即实时过滤当前可见的网站卡片，匹配范围包括**名称、描述、URL**。
  - 命中关键词会在卡片上**高亮**显示。
  - 输入有 200ms 防抖，边打字边出结果。
- **网络搜索**：图标为地球（`Globe`）。点击模式切换图标在两种模式间切换。

#### 3.3.2 选择搜索引擎（网络模式）

网络模式下，可在搜索框内选择搜索引擎：

- 可选来源来自 `search.json`（`externalSources`），默认提供 **Bing / 百度 / GitHub**，默认选中 **百度**。
- 下拉支持键盘导航：`↑` / `↓` 移动、`Enter` 确认、`Esc` 关闭。
- 选中后回车或点击搜索按钮，会在**新标签页**打开对应搜索引擎的结果页。

#### 3.3.3 搜索快捷键

| 快捷键 | 作用 |
| --- | --- |
| `⌘/Ctrl + K` | 聚焦搜索框 |
| `/` | 聚焦搜索框（仅当未处于输入态、焦点不在按钮上、且无模态弹窗时生效） |
| `Esc` | 关闭搜索引擎下拉 / 关闭偏好面板等模态 |

搜索框右侧有**清空按钮**，一键清除当前搜索词。

### 3.4 网站卡片操作

每个网站以卡片（`SiteCard`）形式展示，包含图标、名称、描述。

| 操作 | 方式 | 说明 |
| --- | --- | --- |
| **打开网站** | 点击卡片主体 | 在新标签页打开该网站链接，并记录到「最近访问」 |
| **收藏 / 取消收藏** | 鼠标悬停卡片时点击出现的星标 | 切换收藏状态，并弹出 Toast 提示；收藏内容进入「我的常用」 |
| **复制链接** | 鼠标悬停卡片时点击复制按钮 | 将网站 URL 复制到剪贴板（含降级方案），并弹出 Toast 提示 |

- **图标解析**：卡片图标按优先级解析 —— 仓库内同域文件 `public/favicons/` → 站点自身 `/favicon.ico` → 聚合器降级链（`favicon.im` → `icon.horse` → `faviconextractor` → DuckDuckGo → Google S2）→ 占位图。
- **搜索引擎图标**：若图标加载失败，会回退显示首字母，并带 5 分钟 TTL 缓存。

### 3.5 个人偏好面板

点击顶部「偏好」按钮打开（右侧抽屉，按 `Esc` 关闭）。面板内分三部分。

#### 3.5.1 概览

- 显示当前「收藏」与「最近访问」的计数。

#### 3.5.2 显示密度

- 提供 **舒适 / 紧凑** 两种密度。
- 紧凑模式会降低卡片高度并增加每页展示数量，适合一屏看更多网站。
- 该设置持久化到本机（localStorage 键 `nav_density`）。

#### 3.5.3 备份与恢复

- **导出配置**：将完整个性化数据导出为一个 JSON 文件，文件名形如 `nianming-nav-config-YYYY-MM-DD.json`，内容含 `prefs`（收藏 / 最近访问 / 主题 / 搜索引擎选择）+ `categories` / `links` / `searchConfig` / `settings`。
- **导入配置**：选择此前导出的 JSON 文件恢复。导入会校验格式；在开发模式下会写回磁盘源文件，生产模式下仅应用于当前会话。
- 提示：「偏好仅保存在本机浏览器」——清空浏览器数据或更换设备后，收藏与最近访问不会同步。

#### 3.5.4 数据管理

- **清空收藏**：移除全部收藏。
- **清空最近**：移除全部最近访问记录。

### 3.6 主题与显示

- 点击顶部「主题切换」按钮在**亮色 / 暗色**之间切换。
- 首次访问时**跟随系统**配色（`prefers-color-scheme`）；一旦手动切换，选择会被记忆到本机（localStorage 键 `user-prefs-v1` 的 `theme` 字段），此后不再跟随系统，除非清除该数据。
- 站点主色（accent color）由维护者在 `settings.json` 中配置，对所有用户一致（见 [4.2.4](#424-settingsjson)）。

#### 时间日期组件（仅桌面端头部）

显示当前时间、`YYYY年M月D日`、星期，以及公历中文写法（如「七月十七」）。

> 注意：当前版本显示的是**公历中文日期**，并非农历。

### 3.7 分页与响应式布局

- 当某分类下网站数量超过每页容量时，底部出现分页（`Pagination`）。
- 切换分类、二级分类或搜索词时，自动回到第 1 页（避免结果不足时显示空白页）。
- 响应式：移动端 2 列卡片 + 抽屉式侧边栏；桌面端随宽度自动增加列数（最多 6 列）。

## 第 4 章 维护者 / 部署者手册

本章面向需要**编辑内容、配置站点、部署上线**的人员。

### 4.1 架构与数据模型

技术栈：**Vue 3**（`Composition API` + `<script setup>`）+ **Vite** + **Tailwind CSS v4**（CSS 变量配置，无 `tailwind.config.js`）+ **Pinia** + **Vue Router** + **Vitest** 测试。图标使用 `lucide-vue-next`。

数据流：

```
JSON (src/config/data/) ──懒加载+Zod校验──▶ loadSiteConfig()
        │                                        │
        ▼                                        ▼
  Pinia data store ──静态数据 + getters      Pinia ui store ──交互态
        │                                        │
        ▼                                        ▼
  组件渲染 ◀────────────── App.vue 初始化门控 ──────────────▶ 用户偏好/主题
```

关键目录：

| 路径 | 作用 |
| --- | --- |
| `src/views/HomeView.vue` | 首页视图（唯一前台路由） |
| `src/views/AdminView.vue` | 开发态管理后台视图（仅 DEV） |
| `src/components/layout/` | `Sidebar`、`MainHeader` 布局组件 |
| `src/components/ui/` | `SiteCard`、`UserPrefsPanel`、`TimeDateComponent`、`Pagination`、`SubCategoryMenu` 等 |
| `src/components/admin/` | 管理后台各子面板 |
| `src/stores/` | `data` / `ui` / `theme` / `settings` / `userPrefs` / `admin` 状态 |
| `src/services/faviconService.ts` | 图标解析与缓存服务 |
| `src/config/loadConfig.ts` | JSON 懒加载 + Zod 校验聚合 |
| `src/config/data/` | 四类静态数据 JSON |
| `scripts/` | `fetch-favicons` 图标下载脚本 |
| `plugins/devAdminApi.ts` | 管理后台写盘中间件（仅 DEV） |

> 约定：Vue API 与 `src/components` 下的 SFC 通过自动导入插件注册，**不要**在代码中写显式 `import`；项目**不使用**任何 UI 组件库（Element Plus 已移除）。

### 4.2 数据配置文件说明

所有站点内容位于 `src/config/data/`，共四个文件。修改后无需重启即可在开发模式生效（部分需刷新）；生产构建时需重新 `build`。

#### 4.2.1 categories.json（分类树）

数组，每个分类对象字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | string | 是 | 分类唯一标识，被 `links.json` 的 `categoryId` 引用 |
| `name` | string | 是 | 分类显示名 |
| `icon` | string | 是 | lucide 图标名（如 `bot`、`terminal`） |
| `hidden` | boolean | 否 | 为 `true` 时在前台隐藏该分类 |
| `order` | number | 否 | 排序权重 |
| `subCategories` | array | 否 | 二级分类数组，每项含 `id` / `name` / `icon`（可选） / `order`（可选） |

#### 4.2.2 links.json（网站链接）

数组，每条链接字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | string | 是 | 链接唯一标识（如 `"1001"`） |
| `name` | string | 是 | 网站名称 |
| `url` | string | 是 | 网站地址（建议带 `https://`） |
| `categoryId` | string | 是 | 所属一级分类 `id`，必须存在于 `categories.json` |
| `subCategoryId` | string | 否 | 所属二级分类 `id`，必须存在于对应分类的 `subCategories` |
| `icon` | string | 否 | 自定义图标 URL；缺省则由 favicon 服务自动解析 |
| `pinned` | boolean | 否 | 置顶（如 `true` 则排在该分类前面） |
| `pinnedOrder` | number | 否 | 置顶排序权重 |
| `order` | number | 否 | 普通排序权重 |
| `hidden` | boolean | 否 | 为 `true` 时在前台隐藏该链接 |
| `description` | string | 否 | 网站描述，参与站内搜索匹配 |
| `createdAt` | number | 否 | 创建时间戳 |

> **引用完整性**：加载时会用 Zod 校验 `categoryId` / `subCategoryId` 必须能对应到已有分类，否则配置校验失败、首页无法渲染（见 [5.2](#52-页面空白或提示配置错误)）。

#### 4.2.3 search.json（搜索引擎）

```json
{
  "selectedSourceId": "baidu",
  "externalSources": [
    { "id": "bing",  "name": "Bing",  "url": "https://www.bing.com/search?q=",  "enabled": true, "icon": "..." },
    { "id": "baidu", "name": "百度",  "url": "https://www.baidu.com/s?wd=",     "enabled": true, "icon": "..." },
    { "id": "github","name": "GitHub","url": "https://github.com/search?q=",    "enabled": true, "icon": "..." }
  ]
}
```

| 字段 | 说明 |
| --- | --- |
| `selectedSourceId` | 默认选中的搜索引擎 `id` |
| `externalSources[].id` | 搜索引擎唯一标识 |
| `externalSources[].name` | 显示名 |
| `externalSources[].url` | 搜索 URL，**必须以查询参数结尾**（如 `?q=` / `?wd=`），用户关键词会拼接在后面 |
| `externalSources[].enabled` | 是否在前台可选 |
| `externalSources[].icon` | 图标 URL（可选） |

#### 4.2.4 settings.json（站点设置）

```json
{ "accentColor": "14 165 233" }
```

| 字段 | 说明 |
| --- | --- |
| `accentColor` | 主色，格式为 **RGB 三值空格分隔**字符串（如 `"14 165 233"`）。系统会自动转换为 HSL 并写入 CSS 变量（`--hue-primary` / `--sat-primary` / `--lig-primary`），作为全站主色 |

> 这是**唯一**影响全站视觉的运行时设置项。背景图、卡片样式等设置当前**未实现**，请勿在文档 / 配置中添加相关字段。

### 4.3 图标获取脚本

```bash
npm run fetch-favicons        # 下载所有站点图标到 public/favicons/ 并生成 faviconManifest.json
npm run fetch-favicons --force # 强制重新下载全部图标
```

- 脚本读取 `links.json`，为每个站点抓取图标，保存到 `public/favicons/`，并生成 `src/config/faviconManifest.json` 清单。
- **重要**：每次增删或修改链接后，应运行此脚本刷新图标，以保证线上同源 / 离线可用。
- `public/favicons/` 与 `faviconManifest.json` 均提交到仓库（Cloudflare 可离线服务）。

### 4.4 开发态管理后台

访问 `http://localhost:5173/admin`（**仅 `npm run dev` 时可用**）。包含标签页：

| 标签 | 组件 | 功能 |
| --- | --- | --- |

## 第 5 章 常见问题与异常处理

### 5.1 图标不显示 / 显示占位图

- **原因**：该站点图标在所有解析来源（本地文件 → 站点 `/favicon.ico` → 聚合器链）均失败。
- **处理**：
  1. 确认 `public/favicons/` 是否包含该站点图标；若缺失，运行 `npm run fetch-favicons` 刷新。
  2. 网络受限环境下聚合器链可能失败，属正常现象，可稍后重试或手动放入图标文件。
  3. 图标解析带内存 + localStorage 缓存（版本化 key、30 天 TTL、200 条上限），必要时清除浏览器站点数据后重试。

### 5.2 页面空白或提示配置错误

- **原因**：`links.json` 中 `categoryId` / `subCategoryId` 引用了不存在的分类（Zod 校验失败）。
- **处理**：
  1. 开发模式进入 `/admin` → 「数据检查」标签定位脏数据。
  2. 或直接编辑 `src/config/data/links.json`，修正 / 补齐对应分类 `id`。
  3. 重新加载页面。

### 5.3 搜索无结果 / 站内搜索不匹配预期

- 站内搜索匹配**名称、描述、URL** 三处文本；若网站无 `description` 且名称 / URL 不含关键词，则不会被命中。
- 输入有 200ms 防抖，快速输入时稍候片刻再判断。
- 网络模式下需先选好搜索引擎，回车 / 点按钮才会在新标签打开结果。

### 5.4 收藏 / 最近访问丢失

- 这些数据仅存于本机浏览器（localStorage 键 `user-prefs-v1`）。
- 以下情况会导致丢失：**清除浏览器数据**、**使用隐私模式**、**更换设备 / 浏览器**。
- 可通过偏好面板的「导出配置」提前备份，换设备后「导入配置」恢复（生产模式下导入仅作用于当前会话，不会写回服务器）。

### 5.5 主题没有跟随系统变化

- 一旦手动切换过主题，选择会被记忆并**覆盖**系统跟随行为。
- 恢复「跟随系统」：清除本机 `user-prefs-v1` 中的 `theme` 字段（清除站点数据），重新加载即按系统配色。

### 5.6 管理后台打不开 / 404

- `/admin` 仅在 `npm run dev` 开发模式下注册；`npm run build` 生产构建中已被剔除。
- 若在开发模式仍 404，确认以 `npm run dev` 启动且访问路径为 `/admin`。

### 5.7 新增网站后线上图标仍是旧的

- 改完 `links.json` 后需运行 `npm run fetch-favicons` 重新生成 `public/favicons/` 与 `faviconManifest.json`，再重新 `build` 部署。

### 5.8 搜索引擎下拉键盘无法操作

- 下拉支持 `↑` / `↓` 选择、`Enter` 确认、`Esc` 关闭；若焦点不在搜索框内，请先点击 / 用 `⌘/Ctrl + K` 或 `/` 聚焦搜索框。
